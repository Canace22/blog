这周 Codex 更新了，改了名字和 logo，模型名称也改了，五花八门的名称，让人挺疑惑的，我查了下相关资料争取能说明差别和怎么选择吧。

OpenAI 从 GPT-5.6 开始，把原来 Instant / Thinking / Pro，逐渐改成了三个固定英文名：

  • Sol（太阳）
  • Terra（地球）
  • Luna（月亮）

官方的意思是：以后数字表示代际（5.6），名字表示能力层级，而不是每次换版本都重新叫 Instant、Thinking、Pro，从名称也能看错强弱，Luna->Terra->Sol，我个人觉得这样挺好记的，我现在准备切模型就会想哪个在宇宙中更大。

下面讲讲和几个模型的具体区别和适用场景

一、三个模型的使用场景和区别

  • Luna：最快、最便宜，适合聊天、翻译、写一些简单代码和做批量任务，相当于 Claude Haiku
  • Terra：综合能力最均衡；很多场景已经接近 GPT-5.5，但成本更低，适合写作、日常开发、办公等，相当于 Claude Sonnet
  • Sol：旗舰模型，也是目前 GPT-5.6 家族里最强的一档，适合复杂推理、Agent、科研、架构以及长代码等，相当于 Claude Opus / Fable

二、不知道怎么选模型，无脑上 Ultra 吗？

这次 gpt-5.6 更新，加了一个新的 effort 【Ultra】，听起来感觉就很牛逼，让人忍不住想试试。

于是我也试了下 Sol Ultra，讲讲我觉得不错的点吧。

做事情很主动，模糊的需求可以帮我自动拆分任务，分给多个不同的 subagent(子任务) 去执行，真的能解放双手，叹着茶就把活给干了。

之前我都是要自己写好需求文档，拆好任务，半自动化的让 agent 去干活的，看起来真的能很大的提高效率。

长上下文内容理解更精准，另一个我觉得比较好的地方是，在一个比较复杂的 monorepo 项目中，执行了很久的任务，Sol Ultra 还是能准确的理解信息，在工程化中这是很重要的，之前执行久了复杂长任务，就可能开始有点幻觉了。正如 Codex 负责人 Tibo（重置哥）在推特说的，可以让我们有时间尝试雄心勃勃的任务。

说得这么牛，就没缺点吗？有的，还不少

  • 有不少人反馈，开了 Ultra，Codex 的 app 变得变卡，但是这不是问题，Codex 现在能自己修复自己的 bug 了！对你没听错，就跟 OpenClaw 和 Hermes 的 doctor 命令一样，Codex 也能给自己看病治疗了，但是要花我们自己的 token 给他治病，多少有点不礼貌了。

  • 好的东西花费肯定更高，有朋友说：实测 sol ultra 大概1分钟2-2.5刀，20x 5h限制大概能跑3小时，听起来还是很氪金的。

重置哥 Tibo 也在推特说了：使用GPT-5.6 Sol Medium 作为你的日常主力工具，对于真正棘手的问题则切换到 Extra High。如果你想要绝对最好的效果，并且不怕快速消耗使用量，那么Ultra绝对是一头猛兽。
