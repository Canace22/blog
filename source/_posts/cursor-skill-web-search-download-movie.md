后面看了现在比较新的使用姿势,可能就是接飞书、企业微信等，做远程运维或者是系统操作，想了下，除了远程服务器配置和运维，我好像也没有什么场景可以让 AI 去做的。

有几方面吧，第一点，让 AI 帮我写代码，我平常用工具的 agent 都要花挺多时间对齐，才能达到我想要的效果，换成 IM，我感觉也是一样的，太依赖 LLM 能力了。第二点，做系统或服务器运维，有朋友实践了，告诉我他的体会，说是让他的龙虾帮他在虚拟机上装个Python，花了5块钱，然后还烧了四五十块钱做一个简单的配置，还有一个一开始兴致勃勃装上龙虾准备大干一番的朋友说早删了，龙虾吃光了他的饲料，自己像钓鱼佬，有打窝的钱不如直接去菜市场买鱼。由此可见龙虾不仅烧钱，做事情还不太可控，我没说他做不到，想说的是时间不可控，有时候解决一个问题可能自己停下手中的事在机器上操作一下更快。

扯了一堆，回到正题，我们知道，龙虾实际上就是自然语言+tool/skill，IM 输入自然语言，它可以通过 Agent skill 去操作系统或者 tool，以达到帮我们做事情的目的。除了 IM，其他用 Cursor 也是能做到的，我就想，用 Cursor 的 create-skill 技能去帮我创建 skill，然后去做一些类似龙虾做的事情是不是也可行，于是就有了这个找电影资源并帮我下好的 skill。

cursor Agent 模式，直接让输入【创建技能：通过自然语言对话，从网上搜索影视资源，并自动将用户选定的种子推送到本地进行下载】，skill 的主要流程是这样的：

1. **解析意图**：从对话中提取片名、类型（电影/剧集）、季集、清晰度、偏好（如「要蓝光」「带中字」）。
2. **网上搜索**：用提取的关键词进行 web 搜索（如「片名 种子」「片名 magnet 下载」），不假定具体站点，由搜索结果自然得到链接或磁力。
3. **整理结果**：把找到的条目整理成列表（标题、大小、做种数、magnet 或 .torrent 链接），编号或简要描述，便于用户选择。
4. **用户选定**：让用户通过编号或描述指定要下载的一条或多条。
5. **推送到本地**：根据用户环境，用下面一种或多种方式把选定的资源推到本地下载。

然后关于 Skill 怎么写，我感觉主要思路是这样的吧，总结经验，提炼流程（包括步骤，工具调用等），注明一些容易跑偏的注意事项，比较适合抽成 Skill 的可能是一些需要重复性的固化操作，一些有模板的操作等，比如我写博客用的 hexo，一切都是命令，我每次生成个新笔记或者预览都带点一下左侧的 script 或者终端运行脚本，这种就很适合抽成一个博客操作 skill，让 agent 自行判断我什么时候要做什么，而不用我自己去写命令。
