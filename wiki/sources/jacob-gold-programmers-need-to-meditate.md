# Programmers need to start meditating now（Jake Gold）

## 来源内容摘要

- 过去 20 多年，写代码本身是一种冥想式的专注：长时间处于**心流状态**会抑制**默认模式网络（DMN）**，DMN 负责走神/反思，但也负责反刍式焦虑，所以抑制它能让人心安。作者说如果一周没写代码，会明显感觉不对劲。
- 现在工作方式变成**全天候上下文切换**：同时驱动多个 Agent 会话，早到晚在"切换—审查—合并—重新下指令"之间循环。作者承认产出确实更高，但心流时间大幅减少。
- 结论：既然编程本身不再提供心流，程序员需要**主动找别的方式让大脑安静下来**——要么培养新的、信息密度低的冥想式爱好，要么直接用 Calm、Waking Up 这类 App 做正念练习。

## 用户的解读（Canace，2026-07-14）

> 从"亲手写代码"到"指挥 AI 写代码"，开发者的核心工作正在从深度执行（Deep Execution）转向快速决策与上下文切换（Context Switching）。

1. **"手艺人"属性变弱，"监工"属性变强**：以前写代码像雕刻，长时间盯一段逻辑会屏蔽外界噪音、带来心流式的平静；现在更像导演/监工，同时开多个 Agent 任务，需要在极短时间内审查、合并、纠错、下指令。
2. **高效是"透支注意力"换来的**：这种"极其高效"的体感很可能是高频刺激带来的多巴胺幻觉——以前遇到卡点会停下思考，大脑有喘息时间；现在 AI 瞬间产出、立刻等待决策，大脑长期处于高负荷"判决"状态，这也是不写几行代码却依然脑雾、精神疲惫的原因。
3. **工作之外的"物理隔离"变得必要**：如果工作本身已无法提供心流甚至变成焦虑源，就要主动找替代品——冥想、正念能强迫大脑"关机"；也可以转向手脑协同但信息密度低的活动（练习简单吉他独奏、跟节奏运动、拼图），让 DMN 回到安全的背景音状态。

一句话总结：生产力工具解放了双手，我们反而要更用力地保护大脑。
